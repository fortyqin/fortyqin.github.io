T3

冒泡排序2：给定 T(T≤10)T(T \le 10)T(T≤10) 组询问，每组询问给定整数 n(1≤n≤105)n(1\le n \le 10^5)n(1≤n≤105)、k(0≤k≤n)k(0 \le k \le n)k(0≤k≤n) 和 q(1≤q≤231−1)q(1\le q \le 2^{31}-1)q(1≤q≤231−1)。求有多少个 111 到 nnn 的排列，满足对其进行 kkk 趟冒泡排序（升序）后，所得序列的最长上升子序列长度至少为 n−1n-1n−1。各组询问的答案需对 qqq 取模后输出。

我们要先搞清楚题目类型：计数。对什么计数？对排列计数。

这是我们再熟悉不过的模型了。对于排列计数，我们有以下常见套路：

  1. 选轴（枚举顺序）：我们是按照值域顺序填，还是按照下标顺序填；
  2. 实体（生成方式）：我们是按照绝对位置生成，还是按照相对顺序生成（更有甚者，可以一半按绝对位置，一半按相对顺序）。

当然，分析一道题目应当从这道题本身的性质出发，而不是生搬硬套模型来试错。

就这道题而言，最核心的问题是“冒泡排序”的具体行为是什么，说具体些，就是我们应当如何快速得知一个序列冒泡排序后长什么样子。

为什么要关注这个？因为对于给定的初始排列，我们判定其合法性，完全取决于它冒泡排序后的结果。换句话说，我们需要一个能够快速判定排列是否合法的准则。

现在该怎么思考呢？作为计数问题，可以先弱化成判定问题。如果我们暴力枚举所有的排列，怎么快速看它最终的排列长什么样子？如果我们暴力维护冒泡排序的过程，显然没有什么前途。

现在我们要解决的一个关键子问题是：对于一个给定的长度为 nnn 的排列，如何快速求出进行 kkk 轮冒泡排序后的结果？这时候，我们可以直接引入一个关于冒泡排序的经典结论：

对于长为 n 的序列，进行 k 轮冒泡排序后，前 i 个位置的值所构成的集合，等于初始序列中前 min⁡(i+k,n) 个元素里的前 i 小值构成的集合（这 i 个值在最终序列中不一定升序）。\text{对于长为 } n \text{ 的序列，进行 } k \text{ 轮冒泡排序后，前 } i \text{ 个位置的值所构成的集合，等于初始序列中前 } \min(i+k, n) \text{ 个元素里的前 } i \text{ 小值构成的集合（这 } i \text{ 个值在最终序列中不一定升序）。} 对于长为 n 的序列，进行 k 轮冒泡排序后，前 i 个位置的值所构成的集合，等于初始序列中前 min(i+k,n) 个元素里的前 i 小值构成的集合（这 i 个值在最终序列中不一定升序）。

需要特别注意，下面这个似是而非的结论是假的：进行 kkk 轮冒泡排序后，第 iii 个位置的值等于初始序列中前 min⁡(i+k,n)\min(i+k, n)min(i+k,n) 个元素中的第 iii 小值。

简单证明理解一下上述结论。首先，对于排序后的前 iii 个位置，一定不会包含初始位置在 min⁡(i+k,n)\min(i+k, n)min(i+k,n) 之后的元素，因为在每一轮冒泡排序中，每个元素最多向前移动一个位置。其次，我们可以将每个前缀都看作一个子问题进行数学归纳，发现每一轮冒泡都会把当前前缀的最大值推到当前轮次能到达的最后位置，由此集合相等的结论得证。

我们还需要回忆一个关于冒泡排序的简单推论：进行 kkk 轮冒泡排序后，序列最后的 kkk 个位置一定是排好序的（即全局最大的 kkk 个元素已经就位）。

现在回到这道题目，它对最终排列的要求是：最长上升子序列（LIS）长度不小于 n−1n - 1n−1。这个限制非常紧，意味着最终的排列几乎已经是一个完全升序的排列。

我们可以用数学语言精确描述一下合法最终排列的形态，它只可能是以下三种情况之一：

  1. 整个排列是完全排好序的：

A=(1,2,…,n)A = (1, 2, \dots, n) A=(1,2,…,n)

  1. 存在一个元素 xxx 向后错位移动（即一个较大的数被排在了后面），形态如下：

A=(1,2,…,x−1,x+1,…,x+m,x,x+m+1,…,n)A = (1, 2, \dots, x-1, x+1, \dots, x+m, x, x+m+1, \dots, n) A=(1,2,…,x−1,x+1,…,x+m,x,x+m+1,…,n)

  1. 存在一个元素 xxx 向前错位移动（即一个较小的数被排在了前面），形态如下：

A=(1,2,…,x−m,x,x−m+1,…,x−1,x+1,…,n)A = (1, 2, \dots, x-m, x, x-m+1, \dots, x-1, x+1, \dots, n) A=(1,2,…,x−m,x,x−m+1,…,x−1,x+1,…,n)

但要尤其注意，如果只是相邻两个位置交换，我们单从结果上无法区分它究竟是“向前移动”还是“向后移动”。为了避免重复计算，我们必须在代码实现时人为地钦定一下分类规则。

现在，我们清楚地知道了最后合法的排列必须满足什么条件。它是以列举的方式给出的，也就是说最后满足条件的最终排列形态是非常少的。

为了简便起见，我们的一个良好策略是：先枚举合法的最终形态，然后根据形态去反向填出初始排列，最后再对状态转移进行优化。

前面已经给出了如何快速得知排序后前缀值集合的方法。要得知第 iii 个位置可以填什么值，必须要先知道前 i−1i-1i−1 个位置填了什么。所以，我们最好的做法就是顺着下标去填绝对位置。
那我们是按照数值从小到大填吗？好像不太合适。直接按初始序列的下标从左到右一个一个看吗？好像也不行。
在这道题中，我们应当按照最终序列的顺序去填初始排列。这样做的优势在于：最终序列的前 iii 个位置的值，取决于原始序列的前 i+ki + ki+k 个位置；既然我们已经把前 i−1i-1i−1 个位置用掉的值给删掉了，那么第 iii 个位置就可以直接通过剩下未被覆盖的 k+1k + 1k+1 个候选位置去填即可。这种状态的转移具有很好的独立性。

我们再进行更细致的分析。如果最终是一个完整的排好序的排列，那么在填前 n−kn - kn−k 个位置时，其实每个位置都有 k+1k + 1k+1 种选择；而最后 kkk 个位置，一定已经是排好序的，我们可以直接在最后乘上 k!k!k! 搞定。

否则的话，对于存在一个元素前后挪动的情况，这就意味着初始序列中一定钦定了一些位置是固定的，它们的选择非常有限。基于这些状态限制，我们可以直接写出 O(n2)O(n^2)O(n2) 的递推做法；随后通过直接观察转移式子，利用前缀和等技巧修改一下枚举的变量，就可以将时间复杂度优化到 O(n)O(n)O(n)。

T4

P7966：给定一个包含 2n2^n2n 个节点（编号为 0∼2n−10 \sim 2^n-10∼2n−1）的图，两点有边相连当且仅当其编号的异或值为 222 的整数幂。另给定一棵包含 n(1≤n≤16)n(1 \le n \le 16)n(1≤n≤16) 条边的树 TTT，其节点编号为 0∼n0 \sim n0∼n，边由 nnn 行 x(0≤x≤n)x(0 \le x \le n)x(0≤x≤n) 和 y(0≤y≤n,x≠y)y(0 \le y \le n, x \neq y)y(0≤y≤n,x=y) 给出。要求在图中找出 k=2n−1k=2^{n-1}k=2n−1（若 k<2n−1k<2^{n-1}k<2n−1 有部分分）棵与 TTT 同构且边集两两不交的树。第一行输出 kkk，接下来 kkk 行，每行输出 n+1n+1n+1 个整数，依次表示树 TTT 中节点 0∼n0 \sim n0∼n 在图中对应的节点编号。

我们依然应明确，这道题到底问的是什么。它就是一个构造题，我们要构造若干棵树，它们同构且没有公共边。

对于构造题，我们应当怎么思考：

  1. 构造的对象：若干棵树；
  2. 限制条件：没有公共边；
  3. 特殊性质：部分分，或题目中特别的地方。

注意到，题目中给出的图是隐式的，它不是直接输入的，所以说图应当存在特殊性质。

那关于一张图，我们应当关注它的什么呢？很简单，图是点集和边集构成的，题中已经给出点集，那我们就思考边集到底是什么样子。

一个边集还是太抽象了，我们继续思考它的部件，一个点的所有出边，然后我们发现，不同的点之间右边当且仅当它的二进制位恰好差 111，题目中所有点的编号是小于 2n2^n2n 的所有非负整数啊。这说明什么？所有点的出度都为 nnn 啊。这是什么图？这是 nnn- 正则图。我们还能看出什么，假设我们对每条边 (u,v)(u, v)(u,v) 人为设置一个边权 u⊕vu\oplus vu⊕v，其中 ⊕\oplus⊕ 表示按位异或，我们现在只考虑所有边权为 2k(0≤k<n)2^k(0\le k<n)2k(0≤k<n) 的边，它们不就是一个完美匹配（原图的 222- 因子）吗？我们有常见结论：kkk- 正则二分图一定可以划分为 kkk 个不相交的完美匹配，那我们反过来不也一样吗？（这是我猜的。）关于这张图是二分图，我们可以每条边链接的两个点的 popcount⁡\operatorname{popcount}popcount 的奇偶性不同，直接得到。

现在我们已经知道，原图是一个 nnn- 正则二分图，这是一种常见的正则图（超立方体图），我们可以知道什么东西呢？我们发现原图的边一共有 n×2n−1n \times 2^{n-1}n×2n−1 条（每个点度数都是 nnn，我们直接加起来再除以 222 即可），而我们要构造重构树的边总共需要 n×2n−1n \times 2^{n-1}n×2n−1 条，这说明我们一条边都不能浪费。

我们发现，它的图非常特殊，需要构造的东西也非常特殊，于是我们思考，它构造出来的样子应当非常美观（可能是满足某种相似、平移、对称性）。我们进一步猜测，对于树的每条边，我们可以为它们分别规定一条互不相同的二的幂次的边权，我们直接在钦定一个根，然后根据边权找到它走到的点，即可。

然后我们发现，如果我们都选择 popcount⁡\operatorname {popcount}popcount 为偶数的点当根的话，那我们的边恰好是不重叠的，这道题就被我们做完了，复杂度 O(n2n−1)O(n2^{n-1})O(n2n−1)。

那我们如何尝试去系统性的思考出来了，我们可以有两种角度：群平移和独立集。

群平移是说，我们先选择一个点 000 当作一棵树的根，然后我们根据我们对树任意规定的不重边权构造树，我们直接选择一个平移因子 vvv，给原来的根异或上 vvv 成为新的根即可。我们很难直接去考虑，我们可以看看边不重叠的一个充分不必要条件，那就是边的两个端点的异或值不同。

我们尝试严谨证明一下为什么上面的构造方式可以使得边不重叠：

  1. 首先，对于一棵树内部，因为它们的边权都不同，所以不会有边重叠；
  2. 对于两颗树，它们如果找到了一个权值相同的边，那一定是从根到这条边上的边权异或和是一样的，因为它们的遍历方式肯定相同，但是由于根的权值不一样，这条边两边的端点一定不一样；
  3. 如果发现了两个相同编号的顶点呢？也不用担心，肯定是不会是边权相同的边，那么一定不同。

欸？我们上面的证明好像没有用到根的 popcount⁡\operatorname{popcount}popcount 需要一样啊。哦，我们上面的证明有漏洞，发现如果对于一条边 (u,v)(u, v)(u,v)，若在一棵树上 uuu 是 vvv 的父亲，另一棵树上 vvv 是 uuu 的父亲，好了，我们上面并不能判断出来它们是不是冲突了。

但是如果根的 popcount⁡\operatorname{popcount}popcount 一样的话，对于上面的 case 2，就钦定好了父亲和儿子的 popcount⁡\operatorname{popcount}popcount，好了，完美。

独立集的角度如何理解呢？他不是要求任意两颗树不能有重边吗？也就任意两颗树不能冲突。我们想到什么，我们先建立一个图，节点代表所有可能的树，任意两个点右边当且仅当它们至少公用了原图的一条边。那我们的问题不就是变成了求这样一个图的最大独立集了吗？当然直接求肯定不可能，但我们建出这张图是为了辅助我们思考，帮我把问题转换成了我们熟悉的问题，可能更方便利用一些原本使用的工具。

我们知道对于一般图求最大独立集是 NP-Hard 问题（问一张图是否存在大小至少为 kkk 的独立集是 NP-Complete 问题），所以我们猜测我们建立出来的那张图可能是二分图？
