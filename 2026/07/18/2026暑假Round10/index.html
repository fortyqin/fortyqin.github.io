T2

吃樱桃：给定一棵包含 n(2≤n≤2×105)n(2 \le n \le 2 \times 10^5)n(2≤n≤2×105) 个结点的树，由 n−1n-1n−1 条连接 ui,vi(1≤ui,vi≤n,ui≠vi)u_i, v_i(1 \le u_i, v_i \le n, u_i \neq v_i)ui​,vi​(1≤ui​,vi​≤n,ui​=vi​) 的边组成。每个结点 i(1≤i≤n)i(1 \le i \le n)i(1≤i≤n) 拥有 ki(0≤ki≤m)k_i(0 \le k_i \le m)ki​(0≤ki​≤m) 种属性，分别为 ti,1,ti,2,…,ti,ki(1≤ti,j≤m)t_{i,1}, t_{i,2}, \dots, t_{i,k_i}(1 \le t_{i,j} \le m)ti,1​,ti,2​,…,ti,ki​​(1≤ti,j​≤m)，其中整个系统中共有 m(1≤m≤6)m(1 \le m \le 6)m(1≤m≤6) 种属性。

指定一个结点为根结点 u(1≤u≤n)u(1 \le u \le n)u(1≤u≤n)，边缘结点 v(1≤v≤n)v(1 \le v \le n)v(1≤v≤n) 定义为树中度数为 111 且 v≠uv \neq uv=u 的结点。从 uuu 到 vvv 的主干枝定义为无重复结点的序列 [u=x0,x1,x2,…,xs=v][u=x_0, x_1, x_2, \dots, x_s=v][u=x0​,x1​,x2​,…,xs​=v]，满足 xi(1≤xi≤n)x_i(1 \le x_i \le n)xi​(1≤xi​≤n) 与 xi+1(1≤xi+1≤n)x_{i+1}(1 \le x_{i+1} \le n)xi+1​(1≤xi+1​≤n) 在 i<s(1≤s<n)i < s(1 \le s < n)i<s(1≤s<n) 时通过边相连，且当 i≠j(0≤i,j≤s)i \neq j(0 \le i, j \le s)i=j(0≤i,j≤s) 时 xi≠xjx_i \neq x_jxi​=xj​。一条主干枝所包含的属性集合为该路径上所有结点属性集合的并集。

每次操作可以任意改变一个边缘结点 vvv 拥有的属性种类和数量。

求分别以 1,2,…,n1, 2, \dots, n1,2,…,n 中的每个结点作为根结点 uuu 时，使所有主干枝包含的属性集合完全相同所需的最少操作次数。

对于这道题目，我们依然应先清楚它询问的是什么？是对每个根，求修改叶子节点使得所有叶子节点到根的链的权值按位或相同。

我们的思考方向应当是，先考虑指定其中一个点当根应当怎么做。这时候，我们或许会想，我们可以通过树上 dp 告诉根当前点到链的权值按位或是多少，让子树维护所有根叶路径信息，这应当应当是一个 O(n4m)O(n4^m)O(n4m) 的做法。

具体来说，我们需要维护哪些是我们不关心的，路径上非叶子处已将有值的位是什么，维护的所有根叶路径是同色还是异色。

可惜的是，这种做法是没有前途的，我们貌似没法快速求出所有根的答案，且对一个根求答案的速度也不够快。

我们尝试反思，为了应用换根 dp，我们应当做到什么。我们可以使用换根 dp 的东西通常来说，是把信息加在一起（因为为了使得其具有逆元，可以把一个子树的信息相减），如果我们的 dp 维护的东西是有 if/max/min 等复杂的判断话，我们很难做到减去一个子树的信息。

所以，现在我们的目的是使用换根 dp 维护出足够的信息，然后在根出进行决策（决策只在根处），所以说我们现在需要具体分析题目的性质。

我们再来看看这道题到底问的是什么？问需要修改多少个叶子。它有什么特别的地方？每个叶子如果要修改，我们无论怎么修改，它只需要花费固定的 111 的代价。并且它最终目标是，所有叶子到根路径的权值按位或是相等的。所以啊，只要有叶子没有改，我们就知道了最终它们要变成什么样子。所以说，我们就只需要知道所有终态是否可能到达即可（这种思想，我们前面说到过，终态很少，我们可以直接考虑终态是什么）。

那现在我们的问题就变成了怎么（快速）判断一个终态合法，且对终态的样子计数。怎么判断一个终态合法，我们考虑所有非叶子节点，如果它出现了某一种权值的数，那么最终的终态一定要出现这种权值，反之，我们不关心的这种权值（出现或不出现都可以）。对于终态的这种判断也非常容易理解，可是为什么要这样想呢？因为，我们的叶子是可修改的，我们的非叶子是不可修改的，又因为所有的根叶路径包含了树上的所有节点，我们就可以直接通过这种方式判断一个终态的合法性。其实，这种想法，再想到如果有的叶子没有修改的话，终态就固定了就很容易思考出来。

那么现在我们只需要用树上 dp 维护出每种根叶路径的个数和。然后就可以算出一个答案。为了换根 dp，我们就维护每个子树根到叶子路径的个数，就可以了。

总结我出现的问题，一直没有想到我们应当分析修改的特殊性，如果一个节点是固定，那么它就是终态。我一直在思考使用树上 dp 就判定，这说明思维还是局限了。

欧拉函数游戏：给定一个长度为 n(2≤n≤5×105)n(2 \le n \le 5 \times 10^5)n(2≤n≤5×105) 的排列 ppp，其中 pi(1≤pi≤n)p_i(1 \le p_i \le n)pi​(1≤pi​≤n)。有 q(1≤q≤5×105)q(1 \le q \le 5 \times 10^5)q(1≤q≤5×105) 次询问，每次询问给定区间 [l,r][l, r][l,r]，其中 l,r(1≤l<r≤n)l, r(1 \le l < r \le n)l,r(1≤l<r≤n)。对于每组询问，求区间内两个不同位置元素乘积的最大欧拉函数值，即输出 max⁡l≤i<j≤rφ(pi×pj)\max_{l \le i < j \le r} \varphi(p_i \times p_j)maxl≤i<j≤r​φ(pi​×pj​)，其中 φ(x)\varphi(x)φ(x) 表示不超过 xxx 的正整数中与 xxx 互质的数的个数。

这个题，我们依然先看它问的是什么？多组询问，每次让在区间内选不同的值，使得它们的成绩的欧拉函数值尽可能大。

我们初步认为它是一个数据结构题，不过我们发现这个欧拉函数过于抽象，我们好像没法直接计算它。

我们尝试对式子 φ(nm)\varphi(nm)φ(nm) 进行转化。我们现在必须要做的是，使得欧拉函数的自变量不再是乘积 nmnmnm 的形式。最好将 n,mn,mn,m 独立出来。所以，我们初步的想法是把式子化简成与 φ(n),φ(m)\varphi(n),\varphi(m)φ(n),φ(m) 有关的形式。我们肯定知道欧拉函数是积性函数。如果 gcd⁡(n,m)=1\gcd(n,m)=1gcd(n,m)=1，那么 φ(nm)=φ(n)φ(m)\varphi(nm) = \varphi(n)\varphi(m)φ(nm)=φ(n)φ(m)，欸，这不就独立出来了。

现在我们进一步思考，若 gcd⁡(n,m)>1\gcd(n, m)>1gcd(n,m)>1 该怎么办呢？我们想到欧拉函数的计算式子，若 n=∏i=1kpicin=\displaystyle\prod_{i=1}^k {p_i}^{c_i}n=i=1∏k​pi​ci​，则 φ(n)=n∏i=1kpi−1pi\varphi(n)=n\displaystyle \prod_{i=1}^k \dfrac{p_i-1}{p_i}φ(n)=ni=1∏k​pi​pi​−1​。我们直观的可以发现，我们的 φ(nm)=t⋅φ(n)⋅φ(m)\varphi(nm)=t\cdot\varphi(n)\cdot\varphi(m)φ(nm)=t⋅φ(n)⋅φ(m)，我们发现 t=gcd⁡(n,m)φ(gcd⁡(n,m))t=\dfrac{\gcd(n, m)}{\varphi(\gcd(n, m))}t=φ(gcd(n,m))gcd(n,m)​，我们应当记住这个经典结论：

φ(nm)=φ(n)⋅φ(m)⋅gcd⁡(n,m)φ(gcd⁡(n,m))\varphi(nm)=\varphi(n)\cdot\varphi(m)\cdot\dfrac{\gcd(n, m)}{\varphi(\gcd(n, m))} φ(nm)=φ(n)⋅φ(m)⋅φ(gcd(n,m))gcd(n,m)​

我们发现这个式子使得我们可以不用直接求大值的欧拉函数，所需要用到的欧拉函数值我们都可以预处理。

我们都知道 gcd⁡\gcdgcd 是我们很难维护的，我们通常把 gcd⁡\gcdgcd 弱化成公共因子。而且啊，我们发现，由于我们是最优化的，我们变成公共因子后，它算出来的欧拉函数值只会变小，好了，对答案完全没有影响，完美。

好了，我们现在重申一下问题，对于一组询问 l,rl,rl,r，我们要求的是：

max⁡l≤x<y≤rd∣x,d∣yφ(px)⋅φ(py)⋅dφ(d)\max_{\substack {l\le x<y\le r \\ d\mid x,d\mid y}} \varphi(p_x)\cdot\varphi(p_y)\cdot \dfrac{d}{\varphi(d)} l≤x<y≤rd∣x,d∣y​max​φ(px​)⋅φ(py​)⋅φ(d)d​

这样子，由于给的是一个排列，我们应当想的是枚举 ddd 看看能给哪些询问贡献，这样是调和级数级别的。

这样子，相当于是，我们就是对于每个 ddd，我们激活了一些点，对于每个询问，就找区间里激活点的最大和次大，因为是独立的。这就是独立的优势。

我做这道题时就卡在这里了，因为我觉得贡献的点对可能是由很多的，我就卡住了。

但其实，对于每个 ddd，假设我们激活了 kkk 个位置，那么我们暴力的认为由 O(k2)O(k^2)O(k2) 对是有用的。但时刻记住，这是最优化问题，它有很优秀的性质：我们既可以直接排除一定对最终答案没有贡献的劣质情况，也可以加一下不会使得最终答案变得更优秀的错解。

所以现在我们看看对于一个给定的 ddd，激活了 kkk 个位置，看看有多少个点对是优秀的。我们记 ai=φ(pi)a_i=\varphi(p_i)ai​=φ(pi​)，假设一个点对是 (x,y)(x,y)(x,y)，在欧拉函数值序列 aaa 的区间 [x,y][x,y][x,y] 中，应当保证 ax,aya_x,a_yax​,ay​ 为区间 [x,y][x,y][x,y] 的最大和次大，不然把端点值较小的那个移动到中间最大的那个位置，这样区间是原来区间的一个的子区间，那原来那个区间一定不好啊。这说明什么，对于每个激活的位置 iii，假设 iii 是作为区间次大值，那么一定是在 aaa 上找前面第一个比它大的，找后面第一个比它大的。所以啊，这只有 O(k)O(k)O(k) 个点对是有用的啊。

所以，直接矩形数点即可。还有啊，这个是前后缀的，所以可以直接使用我们优秀的树状数组。

对于数点啊。注意看它的限制啊。就这题而言，我们查询的是 (l,r)(l,r)(l,r)，有的点是 (x,y)(x,y)(x,y)，我们的偏序限制有 l≤x,r≥yl\le x,r\ge yl≤x,r≥y，二位数点，扫描线搞定。

这题是矩形内权值最大的点。有些题可能是求矩形的点权和，那我们好像看着它是四维偏序，注意到，直接容斥变成前缀和，所以就成二维偏序了。
